# Pre-flight Gate — 四步闸门详细步骤

按顺序执行，每一步完成前不得进入下一步。每步完成后必须用一句话向用户口头确认。

## 0.1 收集用户输入（AskUserQuestion 一次性问完）

强制提问 5 项：

1. **行业名称 + 颗粒度**：参考《申万行业分类》/《国民经济行业分类》/GICS，明确一级/二级/三级口径（如"申万二级·光伏设备"）
2. **研究目的**：一级市场投资 / 二级市场投资 / 创业 / 战略 / 学习（决定重点板块权重）
3. **本地资料路径**：用户已上传的资料包路径。不必再单独追问是否有技术报告/财报/研报——资料包就是全部输入
4. **输出路径与文件名**：默认 `./<行业名>-行业研究报告.md`
5. **时间预算**：传递给每个 subagent，影响搜索轮次和报告篇幅
   - 1 小时草稿：搜索 2 轮、硬科技 ≥800 字、KPI 4-6 个
   - 1 天完整版：标准流程
   - 多日深度版：搜索 +1 轮、硬科技 ≥2000 字、KPI 8-12 个

## 0.1b 技术分析模式判定

**默认硬科技模式**：完整技术分析（§2.4 硬科技模式） + Semantic Scholar API 默认启用。

**仅以下行业取消硬科技模式**（主 agent 主动识别并告知用户确认）：

| 行业类型 | 示例 | 技术分析模式 | S2 API |
|---|---|---|---|
| 消费 | 白酒/服装/餐饮/零售/食品/家电 | 基础写法（1-3段简述） | 跳过 |
| 周期/简单制造 | 钢铁/造纸/水泥/纺织/家具 | 基础写法（1-3段简述） | 跳过 |
| 普通服务业 | 旅游/教育/物业/物流 | 基础写法（1-3段简述） | 跳过 |

除此以外的所有行业（含不确定性界行业）一律默认硬科技模式。判定结果向用户确认一句。

## 0.2 检索通道确认

1. **WebSearch（必选）**：系统内置，无需额外配置。用最小成本搜索"今天新闻"测试可用性。
2. **Wind（推荐）**：走 Skill 工具调用 `wind-mcp-skill`，传入简单查询（如"沪深300指数最新行情"）测试。Wind 每日免费赠送 1000 积分，基本够日常行研使用。若未安装或调用失败，告知用户并继续（WebSearch + 公开财报兜底金融数据）。
3. **Semantic Scholar（推荐）**：仅消费/周期/简单制造/普通服务业跳过。Bash 测试连通：
   ```
   curl -s -H "x-api-key: $S2_API_KEY" -o /dev/null -w "%{http_code}" "https://api.semanticscholar.org/graph/v1/paper/search?query=test&limit=1"
   ```
   返回 200 → 可用；否则告知"S2 API 不可达，技术分析仅用 WebSearch"，不阻塞
4. WebFetch 仅用于读取已知 URL，禁止当搜索引擎用

## 0.3 创建工作产物

用 Write 立刻创建：

1. **主报告**：`<output_dir>/<行业名>-行业研究报告.md`，写入骨架
2. **研究笔记**：`<output_dir>/<行业名>-研究笔记.md`，含四段：
   ```markdown
   # <行业名> 研究工作笔记（不对外，仅用于过程追踪）

   ## A. 数据矛盾日志（发现即追加）
   （空，按 §A 表格格式追加）

   ## B. 信息缺口日志（发现即追加）
   （空，按 §B 表格格式追加）

   ## C. 已验证来源池（accepted）
   （空，每次引用前先登记到这里）

   ## D. 检索关键词日志（避免重复检索）
   （空，记录"关键词→命中页数→是否有用"）
   ```
3. **TaskCreate**：把 7 个板块建成 task，按依赖设置 blockedBy：
   - §2.0 → §2.1 → §2.2 → §2.3/§2.4 → §2.5/§2.6 → 核心结论

## 0.4 资料预处理与索引

有本地资料时执行，无则跳过。

### 轻量路径（资料 ≤3 份且总 MD 字数 ≤5 万）
主 agent 直接 Read 全部 MD，手写精简版 `资料索引.md`（每份 1-2 行：文件名 + 主题 + 关键章节）。

### 完整路径（资料 ≥4 份或单份 ≥3 万字）

1. 建立 `_materials/` 目录
2. 格式归一化：
   - PDF/图片 → Skill `2MD` 转 MD
   - HTML → Bash + pandoc 或 `python -m html2text`
   - DOCX → pandoc / `markitdown`
   - MD/TXT → 直接 cp
3. 并发派 Explore subagent 生成索引（每个 MD 一个 subagent）：
   ```
   读取 <materials/xxx.md>，输出≤150字摘要 + 章节目录（## 标题列表，带行号） + 3-6 个关键词标签，返回 JSON：
   {"file":"...", "topic":"...", "summary":"...", "keywords":["..."], "sections":[{"title":"...","line":N}], "word_count":N}
   不要返回原文。
   ```
4. 汇总到 `资料索引.md`

### Subagent 读资料硬规则
- 必须先 Read `资料索引.md`（小文件，≤5K）
- 按相关性挑 1-3 个文件，Read 用 offset+limit 只读对应章节窗口（≤200 行）
- 禁止对 `_materials/` 下文件做无 offset/limit 的整文件 Read
- 索引里无相关材料 → 直接走网络检索
