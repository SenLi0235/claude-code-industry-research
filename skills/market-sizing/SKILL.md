# 市场规模与增长驱动（§2.4）

成长期行业的核心板块。包含多口径测算、情景预测、增长因子拆解。硬科技行业自动激活完整技术分析。

## 高风险板块 — 先交提纲

**先写 6-10 行提纲，交主 agent 给用户确认后再写正文。**

提纲应包含：
- 测算方法选择（需求导向 / 供给导向 / 引用现成报告）及理由
- TAM/SAM/SOM 的边界定义
- 三种情景的核心假设（保守/中性/乐观各一句话）
- 增长驱动因子初步判断
- 数据来源计划

## §2.4.1 多口径定义（当数据源矛盾时强制执行）

不同来源的市场规模打架，通常是因为口径不同。遇到矛盾时定义 2-3 层口径，而不是硬选一个数：

- **广义口径**：包含所有相关品类的最宽统计
- **中义口径**：聚焦本报告研究边界的核心品类
- **狭义口径**：纯行业内最聚焦的子赛道

每层口径写明包含什么/不包含什么、引用哪个来源。

## §2.4.2 强制三口径（Steve Blank《创业者手册》）

- **TAM 潜在市场**：完全饱和时的年销售额
- **SAM 可服务市场**：当前实际市场年销售额
- **SOM 可获得市场**：行业头部玩家能拿到的份额

## §2.4.3 测算方法（必须给计算过程，不能只甩结论）

**优先级 1：直接引用权威数据**（国家统计局、行业协会、券商研报、咨询机构 Frost & Sullivan/IDC/艾瑞/灼识等）；写明发布机构、报告名、年份、页码。

**优先级 2：手动测算**（三选一）：
- **需求导向**：市场规模 = 目标客户数 × 渗透率 × 客均订单量 × 客单价
- **供给导向**：市场规模 = ∑(各玩家产能 × 利用率 × 单价)
- **供需匹配**：交叉验证两侧

每个测算变量必须独立列出来源/假设。测算后必须做**敏感性分析**：列出关键假设 ±20% 的影响。

## §2.4.4 未来 3-5 年规模预测（强制三种情景）

| 情景 | 核心假设 | 规模区间 |
|---|---|---|
| 保守 | 渗透慢增、政策不变、无技术突破 | XX-XX 亿 |
| 中性 | 延续当前趋势 | XX-XX 亿 |
| 乐观 | 渗透加速、政策加码、技术突破 | XX-XX 亿 |

- 每种情景写清前提假设
- 禁止"预计未来 5 年 CAGR 约 15%"无假设来源的单一线性外推

## §2.4.5 增长驱动因素拆解

将行业增速拆解为 3-5 个驱动因子（如：下游需求增长 × 渗透率提升 × 国产替代 × 单价变化）。每个因子列出当前数值、未来趋势、量化依据。判断：增长是靠"量"（需求扩张）还是靠"价"（产品升级/涨价）？

## §2.4.6 技术驱动因子

### 基础写法（仅消费/周期/简单制造/普通服务业，非默认）
简述当前主流技术路线、技术成熟度对成本/性能/渗透率的影响节奏、是否存在迫近的代际切换。1-3 段即可，不需要论文检索。

### 硬科技模式（本 skill 默认模式）

**四源分工**：

| 来源 | 角色 | 典型用法 |
|---|---|---|
| Semantic Scholar API | 学术引擎 | 综述论文建立框架、高引论文定位主流方向、最新论文嗅探新兴方向 |
| WebSearch | 国内验证 | 科普文章、产业落地、公司产品、最新动态 |
| WebSearch | 全球对比 | 海外团队、国际对标、英文技术博客、Google 搜索 |
| Wind 万得 | 产业数据 | 行业产量/产值、上市公司研发投入、产业链财务数据 |
| 用户资料 | 定向补充 | BP/行研/论文 PDF 中的技术判断 |

**必须覆盖的 6 个主题**（顺序和权重灵活安排，融入叙述不写成 6 个独立小节）：

1. **技术全景**（科普风格）：用类比讲清楚"这是什么技术、核心难题是什么"，用 1 个公式锚点点出物理本质。数据源优先用 S2 综述 + WebSearch 科普。

2. **主流路线对比**（强制用表）：2-4 条路线 × 一句话原理 / 关键性能 / 成本 / **适用场景** / 代表企业或团队 / 主要短板。强制含"适用场景"列。

3. **路线深度展开**（科普风格，选取 2-3 条核心）：每条路线按"原理简述 + 材料/方案 + 优点 + 缺点 + 商业化判断"。优缺点必须是具体工程权衡，商业化判断回答"适合什么场景、卡在哪"。

4. **产业落地与核心团队**：谁在用、成熟度（实验室/小批量/量产）、代表产品。哪些团队/公司在做哪条路线。

5. **技术里程碑与趋势**：3-6 个里程碑事件（年份+事件+意义），未来 3-5 年可证伪的演进方向。

6. **路线间关系**：哪些互补（组合使用）、哪些替代（此消彼长），商业化时间表是否同步。

**论文的使用方式**（融入叙述，不单独成节）：
- 综述论文 → 建立路线分类框架
- 高引论文 → 标记为某路线的"代表工作"
- 最新论文（近 1-2 年）→ 嗅探新兴方向
- 论文引用清单附本节末尾表格：paperId / 标题 / 一作+年份 / 被引数 / 关联路线

**S2 API 检索策略**：
- 多 query 覆盖：英文术语 + 缩写 + `survey/review` + 中文术语（WebSearch 互补）
- 过滤：`year=2021-2026`、`minCitationCount=3`、`sort=citationCount:desc`
- 速率：1 RPS，`sleep 1.1s`，429 时 sleep 5s 重试最多 3 次
- 精读触发：摘要+tldr 信息不足时才 WebFetch PDF

**禁止**：
- ❌ 硬填 TRL 等级表/S 曲线/卡脖子矩阵（卡脖子已在 §2.2.4 处理）
- ❌ 罗列完路线就当分析完毕
- ❌ 不标注论文/数据来源
- ❌ "前景广阔""技术领先"等空话
- ❌ 论文单独成节（应融入各主题叙述中）
- ❌ 消费/周期行业强开硬科技模式
