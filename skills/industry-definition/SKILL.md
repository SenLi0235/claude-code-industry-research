# 行业定义与生命周期定位（§2.0 + §2.1）

两个小节共享分类标准和渗透率数据，由同一个 subagent 在一次派工中连写。

## §2.0 行业定义与分类

### 强制输出

**行业定义**：1-2 段说明研究对象，明确包含/不包含什么细分。参考《申万行业分类》。

**行业分类口径**：列出本报告使用的标准（国家《国民经济行业分类》/证监会《上市公司行业分类指引》/申万/GICS），写明对应代码。

**研究边界说明**：本报告覆盖哪些子领域 / 排除哪些。避免"行业"与"赛道"口径混淆，后续所有数据都要对得齐这个边界。

## §2.1 行业生命周期阶段定位

### 强制输出

**生命周期阶段判定**（用渗透率指标）：

| 阶段 | 渗透率 | 特征 |
|---|---|---|
| 导入期 | <15% | 技术路线未定、玩家稀少、需求待教育 |
| 成长期 | 15%-40% | 渗透加速、格局初现、放量驱动 |
| 成熟期 | >40% | 增速放缓、格局固化、存量竞争 |
| 衰退期 | 需求绝对下降 | 替代品挤压、量价齐跌 |

必须列出：
- 渗透率 = 存量用户 / 潜在客户群（写出当前数值 + 数据来源）
- 若渗透率不可得，用替代指标（销量增速曲线/玩家数量变化/政策周期）并说明为何替代

**基于阶段的分析重点**（告诉读者后续侧重哪些板块）：
- 导入期 → 重 TAM 测算（§五）+ 技术路线判断（§五硬科技模式）
- 成长期 → 重市场规模与驱动（§五）+ 竞争格局变化（§四）
- 成熟期 → 重竞争格局（§四）+ 进入壁垒（§三）+ 替代威胁（§六 PEST-T）
- 衰退期 → 重替代品/转型路径（§六 PEST-T）+ 景气度拐点（§七）

**矛盾标注**：若技术仍在快速迭代但渗透率显示"成熟期"，明确点出这一矛盾（可能是统计口径问题，或存在技术路线替代风险）。

## 这个板块不做什么

- ❌ 不展开公司对标（那是 §2.3 的事）
- ❌ 不详细论述市场规模（那是 §2.4 的事）
- ❌ 生命周期判定用渗透率，不用"感觉""行业共识"等主观判断
